# M1 本地执行手册（厂商侧，Windows 电脑跑）

> 目标：把仓库当前代码变成"可对外交付"的加固发布版。全部步骤在你的 Windows 电脑（管理员 PowerShell）完成；本机无 PowerShell，故需你侧实跑。
> 预计耗时：首次约 30–60 分钟（含证书准备）。后续可脚本化。

---

## 0. 前置条件

| 项目 | 要求 |
|------|------|
| 系统 | Windows 10 / 11（x64） |
| PowerShell | 5.1+（`$PSVersionTable.PSVersion` 查看；Win10/11 自带） |
| 权限 | 管理员（右键 PowerShell → 以管理员身份运行） |
| 代码签名证书 | 二选一：① 受信任 CA 购买的**代码签名证书**（推荐 EV 证书过 SmartScreen）；② 自签测试证书（仅本机/内网） |
| 安装包工具 | InnoSetup（官网免费下载安装） |
| 验证环境 | 一台**干净虚拟机**（无授权文件、全新系统），用于回归闭环 |

打开管理员 PowerShell，进入仓库脚本目录：
```powershell
cd <仓库路径>\enterprise-lan-management\scripts
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass   # 仅当前进程放开脚本执行
$PSVersionTable.PSVersion                                   # 确认 >= 5.1
```

## 1. 初始化授权密钥（一次性）

```powershell
.\gen-license.ps1 -Company "示例客户公司" -Edition Pro -Days 365 -OutFile .\company.lic
```

**预期输出**：
```
已生成 RSA-2048 密钥对。
  私钥 -> .\vendor.key.json（请保密、加入 .gitignore、切勿随产品分发）
  公钥已写入 lib-license.ps1。
已签发授权 -> .\company.lic
  版本=Pro  设备上限=50  有效期=365 天  功能=...
  授权唯一ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx  （如需吊销请妥善留存此 ID）
```

**校验**：
- 打开 `lib-license.ps1`，确认顶部 `$script:LicensePublicKeyXml = '...'` **不再是** `'__LICENSE_PUBLIC_KEY__'`（占位符已被真实公钥替换）。
- 确认 `vendor.key.json` 已生成且**未被 git 跟踪**（`.gitignore` 已忽略；可 `git status` 验证不应出现该文件）。

⚠️ 私钥 `vendor.key.json` 是营收命脉，**绝不可提交、不可随安装包分发、不可外发**。丢失则所有已发授权无法续签，须重新签发并换公钥。

## 2. 代码签名

### 2.1 用受信任 CA 证书（推荐，对外发布）
证书已安装到本机证书存储（个人/受信任发布者）时：
```powershell
.\sign-scripts.ps1                                    # 按默认 Subject '厦门金奕鸣科技有限公司' 取证书
```
### 2.2 用 PFX 文件
```powershell
.\sign-scripts.ps1 -PfxPath .\code.pfx -PfxPassword '你的密码'
```
### 2.3 仅本机/内网测试（自签证书）
```powershell
# 生成自签代码证书（仅当前用户）
New-SelfSignedCertificate -Subject 'CN=厦门金奕鸣科技有限公司' -Type CodeSigningCert -CertStoreLocation Cert:\CurrentUser\My
# 然后直接跑 .\sign-scripts.ps1（会从存储取到该证书）
```
> 自签证书在他人机器上会被 SmartScreen/杀软拦截，仅用于本地验证，**对外发布须用受信任 CA 证书**。

### 2.4 校验签名
```powershell
.\sign-scripts.ps1 -VerifyOnly
```
**预期**：全部脚本显示 `VALID`（已签名且有效）。若有 `BAD` 说明签名失效（证书过期/时间戳服务不可达），重签；若有 `UNSIGNED` 说明漏签，重跑签名。

**排错**：
- `未找到可用的代码签名证书` → 确认证书已安装且 Subject 含"厦门金奕鸣科技有限公司"，或用 `-PfxPath`。
- SmartScreen 报"未知发布者" → 换 EV 代码签名证书，或首次发布后累积信誉/加白。
- 时间戳服务不可达 → 换时间戳服务器（脚本默认 `timestamp.digicert.com`，可改 `sign-scripts.ps1`）。

## 3. 生成安装包并签名

1. 安装 **InnoSetup**。
2. 用 **InnoSetup Compiler** 打开 `build/installer.iss` → 菜单 `Build → Compile`。
3. 产物：`build/output/JinNetConnect-Setup-1.0.0.exe`。
4. 对 `setup.exe` 做 Authenticode 签名（推荐用 `signtool.exe`，来自 Windows SDK）：
   ```powershell
   # 示例（signtool 在 Windows SDK 内；也可用 .\sign-scripts.ps1 思路对 exe 签名）
   signtool.exe sign /sha1 <证书指纹> /fd SHA256 `
     /tr http://timestamp.digicert.com /td SHA256 `
     build/output/JinNetConnect-Setup-1.0.0.exe
   ```
   > 若没有 `signtool`，可在 PowerShell 用 `Set-AuthenticodeSignature -FilePath setup.exe -Certificate $cert` 代替。

## 4. 干净虚拟机验证闭环

在**全新虚拟机**中（建议快照还原点，便于反复测）：

| 场景 | 操作 | 预期结果 |
|------|------|----------|
| A. Free 免费版 | 不放 `company.lic`，直接跑 `wizard-gui.ps1` | 可配置；高级功能受限（仅互联+主机列表） |
| B. 付费版激活 | 把 `company.lic` 放入 `scripts/`，跑 `wizard-gui.ps1` | 显示"授权：专业版 … 设备上限=50 有效期至=…" |
| C. 超额阻断 | 用 `MaxDevices=1` 的 lic，在两台机各跑一次 `deploy.ps1` | 第二台提示"授权设备数已达上限" |
| D. 过期阻断 | 签发 `Days=1` 的 lic，等过期或改系统时间后跑 | 提示"授权已过期，请续费/升级" |
| E. 功能门禁 | Free 版下跑 `manager.ps1 -NetPolicy` | 提示"当前授权版本不包含功能 netpolicy" |
| F. 吊销（M2，可选） | 见下方"M2 验证" | 见下 |

> 场景 C 依赖中心 `Mgmt$` 共享（`deploy.ps1` 注册本机前统计已注册设备数）。若未搭中心，可临时把 `MaxDevices` 设小并在同一台重复"撤销+重部署"模拟。

## 附：M2 在线激活 / 吊销 本地自测（可选，验证后续能力）

> 以下验证"在线激活"与"吊销"闭环是否打通，不影响 M1 交付。

**A. 在线激活**
```powershell
# 终端1：启动参考激活服务（需 vendor.key.json）
Copy-Item .\orders.sample.json .\orders.json   # 用示例订单
.\activate-server.ps1 -Port 8765 -OrdersFile .\orders.json

# 终端2：客户端用订单号在线激活
.\gen-license.ps1                                              # 确保公钥已初始化
.\wizard-gui.ps1                                               # 或直接调用函数：
#   Request-OnlineActivation -OrderNo "ORD-2026-0001" -Url "http://localhost:8765/activate"
```
预期：客户端拿到 `company.lic` 并激活成功（等效于离线 lic）。

**B. 吊销**
```powershell
# 1) 记录某 lic 的 Id（签发时打印，或在已生成 lic 的 payload 中读取）
# 2) 厂商侧吊销
.\revoke-license.ps1 -LicenseId "<该Id>" -Reason "客户退款"
# 3) 把生成的 revoked.json 放到客户端 scripts/ 目录（或设环境变量 JIN_REVOKE_URL 指向托管地址）
# 4) 客户端再跑任意入口脚本 → 应提示"该授权已被厂商吊销"
```
