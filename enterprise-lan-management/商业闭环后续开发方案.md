## 2. 本阶段已完成的改动（基线）

### 2.1 缺陷修复
- **P0 数据模型不一致（已修）**：`scripts/deploy.ps1` 上报主机记录补齐 `Gateway / DNS / IsFileServer / RDP_HostOK`，与 `collect-pcinfo.ps1` 字段对齐，`manager.ps1` 资产清单不再显示空白/错值，核心闭环打通。
- **P1 死链接（已修）**：新建 `scripts/wizard-preview.html` 真实可点击的 8 步向导原型；`README-使用说明.md` 目录名 `network/` → `enterprise-lan-management/`，并补列新增文件。
- **P2 编码声明（已修）**：两个向导的 `Save-Config` 去掉显式 `-Encoding UTF8`，继承 `lib-init.ps1` 的 **UTF-8 带 BOM** 默认，兑现文档承诺。

### 2.2 商业闭环基础（新增）
- `scripts/lib-license.ps1`：RSA-2048 验签、有效期 / 设备数 / 功能权益门禁。
- `scripts/gen-license.ps1`：厂商签发工具（生成密钥对、写回公钥、输出 `.lic`）。
- `wizard / deploy / manager / netpolicy / netcheck` 五个入口均接入授权校验。
- 版本分层 **Free(3台/仅基础互联) / Trial(25台/30天全功能) / Pro(50台) / Enterprise(不限+技术支持)**。
- 新增 `商业授权与上线说明.md`、`.gitignore`（私钥 `vendor.key.json` 与 `*.lic` 绝不入库）、`LICENSE` 改为免费/商业双轨。

### 2.3 发布前必做（首次，一次性）
```powershell
cd <仓库>\scripts
.\gen-license.ps1 -Company "客户公司" -Edition Pro -Days 365 -OutFile .\company.lic
```
该命令会自动生成 RSA 密钥对并把**公钥写入 `lib-license.ps1`**（替换占位符）。若不运行，客户端会因"公钥未初始化"拒绝激活。客户将 `company.lic` 放入 `scripts/` 即激活，无需联网。

## 4. 后续商业完整上线开发建议（核心）

> 以下按"商业必要性 + 工程可行性"排序。每项给出**目标、与现有代码的衔接点、建议实现方式**。

### 4.1 授权与计费体系深化
- **授权吊销 / 黑名单（OCSP 风格）**
  - 目标：客户退款/违约后，即使其手持 `.lic` 也无法继续使用。
  - 衔接点：在 `lib-license.ps1` 的 `Get-License` 中增加可选"吊销清单"校验（本地 `revoked.json` 或可选在线接口）。
  - 实现：厂商发布 `revoked.json`（含被吊销 LicenseId 的哈希），客户端启动/续期时拉取并比对；离线场景可嵌入下次签发的 `.lic` 中。
- **权益精细化**
  - 在 `gen-license.ps1` 的 claims 中增加细粒度开关：`auditRetentionDays`、`maxSubnets`、`remoteWipe`、`supportSLA`，由 `lib-license.ps1` 暴露 `Test-Entitlement -Name xxx`。
- **试用转付费**
  - Trial 到期前 N 天，在 `wizard-gui.ps1` / `manager.ps1` 弹窗引导"输入付费 License 或前往购买页"。

### 4.2 订阅 / 续费 / 激活体验
- **在线激活（混合模式）**：保留离线 `.lic`，新增"输入订单号+机器指纹在线换取 `.lic`"通道（`lib-license.ps1` 增加 `Request-OnlineActivation`）。机器指纹取 `ComputerName+主板UUID+SMBIOS` 哈希。
- **浮动授权（Concurrent）**：对 MSP 场景，设备数按"同时在网激活数"计，`deploy.ps1` 注册时向中心 `Mgmt$` 上报心跳，`manager.ps1` 统计并回收离线超期设备名额。
- **自动续费提醒与一键续期**：`lib-license.ps1` 暴露 `Get-LicenseDaysLeft`，临近到期由 `netcheck.ps1` 汇总到控制台。

### 4.3 集中管理控制台（SaaS 化方向）
- **本地 Web 控制台（MVP）**：用 PowerShell 自带 `HttpListener` 暴露只读管控面板（资产地图、体检汇总、授权状态），复用现有 `lib-discovery` / `lib-audit` 数据。
- **中心数据模型升级**：当前资产以 JSON 文件存 `Mgmt$`，建议迁移到 **SQLite（PowerShell 可 P/Invoke 或 `System.Data.SQLite`）** 或嵌入轻量 KV，支撑多子网、历史趋势、审计检索。
- **远程指令总线**：把 `manager.ps1` 的"批量远程执行"抽象为任务队列（中心下发 → 客户端 `deploy.ps1` 拉取执行 → 回写结果），为 SaaS 控制台打基础。

### 4.4 安装分发与自动更新
- **安装包**：打包 `scripts/` + 依赖检查（PowerShell 5.1+/Win10+）为 MSIX 或 InnoSetup，首次运行引导 `gen`/`activate`。
- **自动更新**：`lib-init.ps1` 增加 `Update-Check`（比对版本清单 `versions.json`），支持差量拉取并校验签名哈希；更新包同样走 RSA 验签，防止供应链投毒。

### 4.5 安全加固
- **代码签名**：对 `.ps1` 用 Authenticode（厂商代码证书）签名，`lib-init.ps1` 启动时校验脚本完整性，防止被客户/第三方篡改后冒用授权逻辑。
- **防调试 / 防提取**：`lib-license.ps1` 公钥与验签逻辑做轻度混淆；`.lic` 增加"绑定机器指纹"可选模式（默认关闭以兼容重装）。
- **最小权限**：当前 `deploy.ps1` 提权到管理员，建议收缩为"仅部署所需"的细粒度权限，并记录提权审计。

### 4.6 遥测、审计与合规商业化
- **可选遥测（客户授权后）**：匿名上报版本/规模/功能使用率，驱动产品决策；在 `LICENSE` 与向导首屏明确告知并可关闭。
- **审计报告导出**：`lib-audit.ps1` 增加 PDF/Excel 导出，作为合规交付物（金融/政企客户刚需）。
- **留存策略随权益**：审计留存天数按 `auditRetentionDays` 权益控制。

### 4.7 多租户 / MSP 支持
- `company-config.json` 增加 `TenantId`；中心数据按租户隔离；`gen-license.ps1` 支持批量签发与渠道（代理）折扣标记。

### 4.8 客户支持、文档与 SDK
- **支持入口**：`wizard-gui.ps1` 增加"联系厂商/提交日志"按钮，自动打包 `lib-audit` 日志与脱敏配置。
- **开发者文档 / SDK**：开放 `lib-*` 接口契约，供客户二开；提供示例脚本与 Postman/OpenAPI（若落地 4.3 控制台）。

## 5. 开发路线 / 里程碑

| 里程碑 | 目标 | 关键交付 | 建议周期 |
|--------|------|----------|----------|
| **M1 加固发布版** | 当前闭环可对外交付 | 代码签名 + 安装包 + 文档定稿 + 首次 `gen-license` 跑通 | 2 周 |
| **M2 授权运营化** | 可计费可续费 | 在线激活 + 吊销名单 + 试用转付费引导 | 3 周 |
| **M3 集中控制台** | 降低交付/运维成本 | 本地 Web 控制台 + SQLite 中心 + 任务队列 | 6 周 |
| **M4 安全与合规** | 打开政企市场 | 防篡改校验 + 审计导出 + 可选遥测 | 4 周 |
| **M5 SaaS/多租户** | 规模化营收 | 在线控制台 + 多租户 + 自动更新 | 8 周 |
| **M6 商业化完整上线** | 全要素闭环 | 订阅计费 + 支持工单 + SDK + 渠道体系 | 持续 |

> 优先级建议：M1 → M2 → M3 是"能卖、能收钱、能省运维"的最短路径；M4/M5/M6 为规模扩张准备。

## 7. 技术债与 TODO

- **未实跑验证**：本阶段改动为静态一致性审查 + 标准 .NET RSA 验签模式，本机无 PowerShell 环境，**首次签发后务必在 Win 环境实跑 Free/Trial/Pro 各一遍**。
- **中心存储弱**：资产/审计当前为 JSON 文件，规模上来后需 SQLite/KV（见 4.3）。
- **门禁粒度**：`manager.ps1` 默认动作映射为 `remotemgmt`，需按实际售卖权益再细分。
- **吊销机制缺失**：上线前至少落地"本地 revoked.json"最小版（见 4.1）。
- **机器指纹激活**：在线激活通道需定义稳定的指纹算法，避免重装即失活引发客诉。
