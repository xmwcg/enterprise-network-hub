- **P2 编码声明（已修）**：两个向导的 `Save-Config` 去掉显式 `-Encoding UTF8`，继承 `lib-init.ps1` 的 **UTF-8 带 BOM** 默认，兑现文档承诺。

### 2.2 商业闭环基础（新增）
- `scripts/lib-license.ps1`：RSA-2048 验签、有效期 / 设备数 / 功能权益门禁。
- `scripts/gen-license.ps1`：厂商签发工具（生成密钥对、写回公钥、输出 `.lic`）。
- `wizard / deploy / manager / netpolicy / netcheck` 五个入口均接入授权校验。
- 版本分层 **Free(3台/仅基础互联) / Trial(25台/30天全功能) / Pro(50台) / Enterprise(不限+技术支持)**。
- 新增 `商业授权与上线说明.md`、`.gitignore`（私钥 `vendor.key.json` 与 `*.lic` 绝不入库）、`LICENSE` 改为免费/商业双轨。

### 2.3 发布前必做（首次，一次性）
```powershell
cd <仓库>\scripts
.\gen-license.ps1 -Company "客户公司" -Edition Pro -Days 365 -OutFile .\company.lic
```
该命令会自动生成 RSA 密钥对并把**公钥写入 `lib-license.ps1`**（替换占位符）。若不运行，客户端会因"公钥未初始化"拒绝激活。客户将 `company.lic` 放入 `scripts/` 即激活，无需联网。

## 4. 后续商业完整上线开发建议（核心）

> 以下按"商业必要性 + 工程可行性"排序。每项给出**目标、与现有代码的衔接点、建议实现方式**。

### 4.1 授权与计费体系深化
- **授权吊销 / 黑名单（OCSP 风格）**
  - 目标：客户退款/违约后，即使其手持 `.lic` 也无法继续使用。
  - 衔接点：在 `lib-license.ps1` 的 `Get-License` 中增加可选"吊销清单"校验（本地 `revoked.json` 或可选在线接口）。
  - 实现：厂商发布 `revoked.json`（含被吊销 LicenseId 的哈希），客户端启动/续期时拉取并比对；离线场景可嵌入下次签发的 `.lic` 中。
- **权益精细化**
  - 在 `gen-license.ps1` 的 claims 中增加细粒度开关：`auditRetentionDays`、`maxSubnets`、`remoteWipe`、`supportSLA`，由 `lib-license.ps1` 暴露 `Test-Entitlement -Name xxx`。
- **试用转付费**
  - Trial 到期前 N 天，在 `wizard-gui.ps1` / `manager.ps1` 弹窗引导"输入付费 License 或前往购买页"。

### 4.2 订阅 / 续费 / 激活体验
- **在线激活（混合模式）**：保留离线 `.lic`，新增"输入订单号+机器指纹在线换取 `.lic`"通道（`lib-license.ps1` 增加 `Request-OnlineActivation`）。机器指纹取 `ComputerName+主板UUID+SMBIOS` 哈希。
- **浮动授权（Concurrent）**：对 MSP 场景，设备数按"同时在网激活数"计，`deploy.ps1` 注册时向中心 `Mgmt$` 上报心跳，`manager.ps1` 统计并回收离线超期设备名额。
- **自动续费提醒与一键续期**：`lib-license.ps1` 暴露 `Get-LicenseDaysLeft`，临近到期由 `netcheck.ps1` 汇总到控制台。

### 4.3 集中管理控制台（SaaS 化方向）
- **本地 Web 控制台（MVP）**：用 PowerShell 自带 `HttpListener` 暴露只读管控面板（资产地图、体检汇总、授权状态），复用现有 `lib-discovery` / `lib-audit` 数据。
- **中心数据模型升级**：当前资产以 JSON 文件存 `Mgmt$`，建议迁移到 **SQLite（PowerShell 可 P/Invoke 或 `System.Data.SQLite`）** 或嵌入轻量 KV，支撑多子网、历史趋势、审计检索。
- **远程指令总线**：把 `manager.ps1` 的"批量远程执行"抽象为任务队列（中心下发 → 客户端 `deploy.ps1` 拉取执行 → 回写结果），为 SaaS 控制台打基础。

### 4.4 安装分发与自动更新
- **安装包**：打包 `scripts/` + 依赖检查（PowerShell 5.1+/Win10+）为 MSIX 或 InnoSetup，首次运行引导 `gen`/`activate`。
- **自动更新**：`lib-init.ps1` 增加 `Update-Check`（比对版本清单 `versions.json`），支持差量拉取并校验签名哈希；更新包同样走 RSA 验签，防止供应链投毒。

### 4.5 安全加固
- **代码签名**：对 `.ps1` 用 Authenticode（厂商代码证书）签名，`lib-init.ps1` 启动时校验脚本完整性，防止被客户/第三方篡改后冒用授权逻辑。
- **防调试 / 防提取**：`lib-license.ps1` 公钥与验签逻辑做轻度混淆；`.lic` 增加"绑定机器指纹"可选模式（默认关闭以兼容重装）。
- **最小权限**：当前 `deploy.ps1` 提权到管理员，建议收缩为"仅部署所需"的细粒度权限，并记录提权审计。

### 4.6 遥测、审计与合规商业化
- **可选遥测（客户授权后）**：匿名上报版本/规模/功能使用率，驱动产品决策；在 `LICENSE` 与向导首屏明确告知并可关闭。
- **审计报告导出**：`lib-audit.ps1` 增加 PDF/Excel 导出，作为合规交付物（金融/政企客户刚需）。
- **留存策略随权益**：审计留存天数按 `auditRetentionDays` 权益控制。

### 4.7 多租户 / MSP 支持
- `company-config.json` 增加 `TenantId`；中心数据按租户隔离；`gen-license.ps1` 支持批量签发与渠道（代理）折扣标记。

### 4.8 客户支持、文档与 SDK
- **支持入口**：`wizard-gui.ps1` 增加"联系厂商/提交日志"按钮，自动打包 `lib-audit` 日志与脱敏配置。
- **开发者文档 / SDK**：开放 `lib-*` 接口契约，供客户二开；提供示例脚本与 Postman/OpenAPI（若落地 4.3 控制台）。

## 8. M1 实现说明（仓库已含脚本）

M1 目标：当前闭环**可对外交付**。本仓库已落地以下加固发布版要素：

| 要素 | 文件 | 说明 |

| 安装包模板 | `build/installer.iss` | InnoSetup 脚本，打包 `scripts/` 并调用 `install.ps1`；建议对 `setup.exe` 再签名 |

### M1 发布流程（厂商侧 SOP）
1. **初始化授权密钥（一次性）**：`.\gen-license.ps1 -Company 客户 -Edition Pro -Days 365 -OutFile .\company.lic`（公钥写入 `lib-license.ps1`）。
2. **代码签名**：用受信任 CA 的代码签名证书运行 `.\sign-scripts.ps1`（推荐 EV 证书以快速通过 SmartScreen）。
3. **生成安装包**：InnoSetup 编译 `build/installer.iss`，并对 `setup.exe` 做 Authenticode 签名。
4. **交付**：将 `setup.exe` + 对应 `company.lic` 交给客户；Free 版可无 lic 直接安装。
5. **回归**：干净虚拟机验证 激活 → 门禁 → 超额/过期阻断。

## 10. M3 实现说明（仓库已含脚本）

M3 目标：**降低交付与运维成本**。把 `manager.ps1` 的"逐台 WinRM 手工执行"升级为"控制台下发 → 终端 agent 拉取执行 → 回写结果"的任务队列，并叠加一个本地 Web 仪表盘。本仓库已落地：

| 要素 | 文件 | 说明 |
|------|------|------|
| 控制台共享库 | `scripts/lib-console.ps1` | 设备注册表 + 任务队列（claim/report）+ 兼容读取 `Mgmt$` 资产/审计/策略；零依赖 JSON 文件存储 |
| 控制台服务 | `scripts/console.ps1` | `HttpListener` 提供 Web 仪表盘（`console.html`）+ REST API；启动需 `remotemgmt` 授权；写操作需管理员令牌（启动时打印） |
| 终端代理 | `scripts/agent.ps1` | 注册 → 轮询任务 → 本地执行（复用 `netpolicy.ps1`/`netcheck.ps1` 函数，保证与 `manager.ps1` 行为一致）→ 上报结果 |
| 仪表盘前端 | `scripts/console.html` | 设备/资产/任务三视图 + 任务下发表单，8 秒自动刷新 |
| 文档 | README「集中管理控制台」小节；`.gitignore` 排除 `console-data/` 与 `*.token` | — |

### 任务类型
- `command`：本地执行一段 PowerShell（Payload.command），捕获输出。
- `netpolicy`：本地应用上网策略（Payload.policy=allow\|deny，Payload.hardcut）。
- `netcheck`：本地执行网络体检，回传 Score/Fail/Warn。

### M3 验证（厂商侧 SOP）
1. 控制台：` .\console.ps1 -Lan` → 记录打印的令牌。
2. 终端：` .\agent.ps1 -ConsoleUrl http://<管理机IP>:8080 -Token <令牌>`。
3. 浏览器开 `http://localhost:8080/` → 设备列表出现该终端 → 建 `netcheck` 任务（目标留空）→ 任务状态变 `completed` 且结果可见。

## 11. M3.5 实现说明（C盘瘦身 / 系统垃圾清理）

M3.5 目标：**解决"无论 C 盘多大都会用一段时间就满"的普遍痛点**，提供单机 + 企业批量下发的系统垃圾清理能力，且严格保证"只删系统产生的垃圾、不碰用户/企业生产数据"。本仓库已落地：

| 要素 | 文件 | 说明 |
|------|------|------|
| 清理核心库 | `scripts/lib-diskclean.ps1` | 硬编码**白名单**（类别→路径模板→风险等级）；安全扫描（只读零删除）、容量统计、备份到 D:/U盘、删除（仅白名单+路径校验）、删除后系统健康检查、自动还原、待确认任务文件生成 |
| 单机入口 | `scripts/diskclean.ps1` | **CLI 向导 + 简易图形（WinForms）双形态**：无参数=图形勾选（适合小白）；逐项中文说明+风险告知；备份盘选择（D:/U盘）；确认后才删除；休眠文件等高风险项默认不出现，仅高级选项提供 |
| 终端代理接入 | `scripts/agent.ps1` | 新增 `diskclean` 任务类型：终端**仅生成清理预览并写待确认文件、绝不静默删除**；本地用户需运行 `diskclean.ps1` 确认后执行（满足"每台终端必须本地弹确认"） |
| 控制台前端 | `scripts/console.html` | 任务类型下拉新增 `diskclean`，含 payload 示例 |
| 授权权益 | `scripts/lib-license.ps1` | 各版本 Features 新增 `diskclean`（**Free 版也开放**，利于获客）；`diskclean.ps1` 本身不强制门禁 |

### 安全设计（对应"100% 安全"边界，诚实表述而非承诺绝对零风险）
1. **仅白名单**：可删项全部来自 `lib-diskclean.ps1` 写死的 `$AllowList`（`updcache/systmp/usertmp/recycle/browsercache/wer/thumbcache/deliveryopt/winsxs/windowold/hiberfil`），绝不"全盘找大文件就删"；`AppData` 仅精确匹配缓存子目录，绝不整目录扫描。
2. **用户可选删除/跳过**：扫描结果逐项中文列出，图形界面可勾选/跳过；默认仅勾选非高级且有占用的项。
3. **中文明文 + 风险告知**：每项含中文名称、路径、预估大小、风险等级（低/中/高）与说明。
4. **一键备份到 D: 或 U 盘**：删除前先把待删项复制到 `备份盘\JinDiskClean_Backup\<时间戳>\`，并生成 `restore.ps1` + `恢复说明.txt`。
5. **防误删 & 可恢复**：双击 `restore.ps1` 即可原样还原；删除后 `Test-SystemHealth` 检查关键服务/进程，异常自动从备份还原。
6. **确认后才删除**：扫描 → 人工确认 → 才出现"执行删除"；高风险项（如 `hiberfil.sys`）默认不出现，仅在高级选项提供并强提示。
7. **不碰红线（代码层强制）**：`pagefile.sys` 只提示不处理；所有删除路径经 `Assert-SafePath` 校验落在允许根目录内，否则跳过。

### diskclean 任务 Payload（企业批量下发）
```json
{ "backupDrive":"D:", "categories":["updcache","systmp","recycle","browsercache","wer","thumbcache","deliveryopt","winsxs"], "advanced":false }
```
> ⚠ 终端收到任务后**不会自动删除**，而是生成本地预览文件并提示登录用户；必须本地运行 `diskclean.ps1` 查看、勾选、确认后才执行。

2. 勾选若干项 + 备份到 D: → 执行 → 验证 D: 备份存在、`C:` 空间释放、系统可正常重启。
3. 高风险演示：勾选 `hiberfil`（高级）→ 验证强提示与 `恢复说明.txt` 可用。
