## 1. 商业闭环总览

本软件不是"一次性交付的脚本集"，而是一套**可计量、可管控、可续费**的产品。闭环如下：

```
[厂商签发]  gen-license.ps1 ──(RSA-2048 私钥签名 .lic)──► 客户拿到 company.lic
                                                              │
[客户激活]  把 company.lic 放入 scripts/                      ▼
                                                              │
[运行时校验] 向导/部署/管控启动时自动校验：签名 ✓ 有效期 ✓ 设备数 ✓ 功能权益 ✓
                                                              │
[能力开放]  按版本开放功能（互联 / 资产清单 / 上网管控 / 网络体检 / 远程管控）
                                                              │
[使用上报]  授权与操作审计写入中心 Mgmt$\audit（厂商可对账）
                                                              │
[续费驱动]  到期 / 超设备数 → 功能阻断 → 客户联系厂商续费升级 ──► 回到[厂商签发]
```

闭环的关键：**没有有效授权 → 高级功能不可用 → 驱动续费**，从而形成持续营收，而非一次性卖断。

## 2. 版本与权益模型

| 版本 | 设备上限 | 功能权益 | 适用 |
|------|----------|----------|------|
| **Free（免费版）** | 3 | 基础互联 + 主机列表 | 个人/极小团队试用 |
| **Trial（试用版）** | 25（30天） | 全部功能 | 评估期客户 |
| **Pro（专业版）** | 50 | 全部功能 | 中小企业 |
| **Enterprise（企业版）** | 不限 | 全部功能 + 技术支持 SLA | 大型/多分支 |

功能权益字段（`Features`）：
- `interconnect`：自动发现 + 自部署（文件共享/远程桌面/工作组/账号）
- `list`：列出已发现主机
- `inventory`：全网资产收集并导出 Excel/CSV
- `netpolicy`：按主机允许/禁止上网 + 网关黑名单导出
- `netcheck`：网络体检与健康报表
- `remotemgmt`：批量远程执行 / 推送文件 / 执行脚本
- `support`：厂商技术支持 SLA（企业版）

## 3. 厂商如何签发授权（供给端）

在**厂商内部、可信环境**运行 `gen-license.ps1`（首次运行会自动生成 RSA-2048 密钥对）：

```powershell
cd <仓库>\scripts
# 仅首次：自动生成密钥对，私钥存 vendor.key.json（务必保密、已 .gitignore）
# 签发一个专业版授权，有效期 365 天
.\gen-license.ps1 -Company "某某科技有限公司" -Edition Pro -Days 365 -OutFile .\company.lic
# 签发 30 天试用
.\gen-license.ps1 -Company "评估客户" -Edition Trial -Days 30 -OutFile .\trial.lic
# 签发不限设备的企业版（永久）
.\gen-license.ps1 -Company "大客户" -Edition Enterprise -Days 0 -OutFile .\ent.lic
```

产出 `company.lic`（JSON：`payload` 为 Base64 声明，`signature` 为 RSA 签名）。通过安全渠道交付客户。

> ⚠️ 安全红线：`vendor.key.json`（私钥）**绝不能**入库或随产品分发。一旦泄露，需更换密钥对并重新签发全部客户授权。

## 4. 客户如何激活（需求端）

客户把收到的 `company.lic` 放到 `scripts/` 目录（或与 `company-config.json` 同目录），
后续运行向导/部署/管控时自动完成激活校验，无需联网。

未放授权文件时，工具按 **Free 版** 运行（3 台设备、仅基础互联），保证开箱可用，
但高级管控（资产清单/上网管控/体检/远程管控）会被门禁拦截并提示升级——这正是转化的入口。

## 5. 运行时门禁（闭环执行点）

| 触发动作 | 所需权益 | 无权益时行为 |
|----------|----------|--------------|
| `deploy.ps1` 部署 | `interconnect`（全版本含） | 始终允许；超设备数则阻断 |
| `manager.ps1 -ListOnly` | `list` | Free 可用 |
| `manager.ps1 -CollectInventory` | `inventory` | 报错提示升级 |
| `manager.ps1 -NetPolicy*` | `netpolicy` | 报错提示升级 |
| `manager.ps1 -NetCheck* / -NetReport` | `netcheck` | 报错提示升级 |
| `manager.ps1`（默认远程命令 / `-PushFile` / `-ScriptFile`） | `remotemgmt` | 报错提示升级 |

设备数上限在 `deploy.ps1` 注册本机时校验：已注册数 + 1 > 上限则阻断部署并提示升级。

## 6. 安全设计

- **防伪造**：授权用 RSA-2048 签名，公钥内嵌客户端（`lib-license.ps1`），私钥仅厂商持有。
  客户端校验失败即视为非法，无法激活。
- **防篡改**：声明与签名分离，任何对 `.lic` 内容的修改都会使验签失败。
- **过期即失效**：超过 `Expiry` 自动阻断，驱动续费。
- **密钥隔离**：签发与验证分离，客户拿不到私钥，无法自行伪造授权。
- **可审计**：授权状态与所有操作写入中心 `Mgmt$\audit`，厂商可远程对账与溯源。
