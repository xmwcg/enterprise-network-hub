# 公司局域网电脑互联互通 · 实施方案（自动发现 + 自部署版）

> 范围：单办公室局域网（同一交换机/路由器下）
> 目标：文件/资源互访 + 远程桌面控制 + 统一管控（综合）
> 形态：**脚本在每台电脑上自动检测本机、自动扫描采集公司其他电脑，并基于采集结果直接完成部署**——无需人工逐台粘贴参数，也无需预先写死主机清单
> 生成日期：2026-07-13（v2 自动发现版）

---

## 1. 设计理念

上一版要求你先收集每台电脑的参数再交给我生成静态配置。本版改为**交付可自运行的工具**：脚本运行时自带"感知"能力——

- **自动检测当前电脑**：主机名、IP、系统版本、是否支持 RDP 主机，全部即时读取，不靠手填。
- **自动采集公司电脑**：对局域网网段做存活扫描（ICMP + TCP/445 + TCP/3389），解析主机名，得出全网对端清单。
- **根据采集结果直接部署**：依据发现到的对端，自动判定文件服务器角色、映射共享盘、把 TrustedHosts 设为发现到的对端、把本机信息与对端清单上报到中心，并开启 WinRM 供统一管控。

你只需做一次**最小配置**（`company-config.json`：工作组名、管理账号、共享路径、文件服务器策略），然后在每台电脑上跑同一个 `deploy.ps1`。

## 2. 总体架构与数据流

```
每台电脑运行 deploy.ps1（管理员）
 ├─ 自动检测本机 ──────────────► 主机名 / IP / 系统 / RDP能力
 ├─ 载入 company-config.json ──► 工作组 / 账号 / 共享 / 文件服务器策略
 ├─ 应用基线 ─────────────────► 专用网络 + 发现 + SMB + RDP(NLA) + 工作组 + 管理账号
 ├─ 自动采集对端 ─────────────► 子网扫描 → 全网清单(peers)
 ├─ 判定文件服务器角色 ───────► AUTO：已有 CompanyShare 则作客户端，否则本机认领
 └─ 直接部署 ─────────────────► 映射共享盘 / TrustedHosts=对端 / 上报本机+清单到 Mgmt$ / 启用 WinRM

                 ┌────────────────────────────────────┐
                 │  文件服务器 PC-FILE                  │
                 │   CompanyShare (公共共享)            │
                 │   Mgmt$ (隐藏)                       │
                 │     ├─ hosts\<PC>.json  (各机自报)    │
                 │     └─ inventory\<PC>_peers.json     │
                 └────────────────────────────────────┘
                          ▲ 管理机运行 manager.ps1
                          读取 Mgmt$\hosts，批量远程执行
```

## 3. 文件清单

| 文件 | 作用 |
|------|------|
| `company-config.json` | **管理员一次性填写**的最小配置 |
| `lib-discovery.ps1` | 发现库：子网枚举 / 端口探测 / 对端发现（被其他脚本引用） |
| `lib-audit.ps1` | 审计库：统一写 JSONL 溯源日志（本地 + 中心 Mgmt$） |
| `deploy.ps1` | **核心**：自动检测 + 自动采集 + 直接部署（每台电脑运行） |
| `discover.ps1` | 仅发现并打印全网对端（部署前预览，只读不改） |
| `manager.ps1` | 统一管控：读取中心清单，批量远程执行 / 上网管控 / 网络体检 |
| `netpolicy.ps1` | 上网管控：按主机允许/禁止上网（客户端防火墙）+ 网关黑名单导出 |
| `netcheck.ps1` | 网络体检：只读检查本机/远端网络健康并集中汇总 |
| `collect-pcinfo.ps1` | 单台快速自检（可选，输出一行 JSON） |

## 4. 前置条件与假设

1. 所有电脑 **Windows 10/11 专业版或企业版**（家庭版不支持 RDP 主机，见第 8 节）。
2. 同一局域网、同一网段（如 `192.168.1.0/24`），互相可通。
3. 执行脚本需**本地管理员权限**。
4. 默认工作组模式（无 AD 域）；统一管控以 WinRM + 中心清单方式轻量实现。

## 5. 运行步骤

**步骤 0｜填一次配置**
编辑 `company-config.json`：工作组名、管理账号名、共享路径、文件服务器策略（`AUTO` 或填主机名）。密码不在配置里。

**步骤 1｜预览发现（可选但推荐）**
在任一台电脑以管理员运行：
```powershell
.\discover.ps1
```
确认脚本能"看到"网内其他电脑。若发现不全，多为防火墙拦截 ICMP/TCP 探测，可改用 `company-config.json` 的 `Discover` 指定范围（如 `"192.168.1.1-254"`）。

**步骤 2｜部署文件服务器**
在准备做文件服务器的那台电脑运行（显式指定更稳妥，避免与 AUTO 认领竞态）：
```powershell
.\deploy.ps1 -FileServer
```
它会创建 `CompanyShare` 与隐藏的 `Mgmt$`，并上报自身。

**步骤 3｜逐台打通其余电脑**
在其余每台电脑运行：
```powershell
.\deploy.ps1
```
脚本自动检测本机、扫描发现对端、判定自己为客户端、映射 `\\<文件服务器>\CompanyShare` 到 `S:` 盘、把 TrustedHosts 设为发现到的对端，并上报自身与对端清单。
> 若某台重启后生效（加入工作组），脚本会提示。

**步骤 4｜统一管控**
在管理机运行（读取中心清单批量操作）：
```powershell
.\manager.ps1                 # 默认取各机系统版本与开机时间
.\manager.ps1 -ListOnly       # 仅列出已发现主机
.\manager.ps1 -FileServerHost PC-FILE -Command { Get-Service Spooler | Select-Object Status }
```

## 6. "自动检测 / 自动采集 / 直接部署"分别做了什么

- **自动检测本机**：读取 `$env:COMPUTERNAME`、IPv4 地址、`Win32_OperatingSystem`、版本（`WindowsEditionId`），据此判断能否作 RDP 主机。
- **自动采集对端**（`Find-Peers`）：按配置范围（默认本机子网）枚举 IP；对每个地址用 ICMP（ping）、TCP/445（SMB）、TCP/3389（RDP）任一种存活即记为在线；通过 DNS 反查主机名；输出 `{IP, Name, RDP, SMB, IsSelf}` 清单。
- **直接部署**：
  - 文件服务器角色：AUTO 下若已存在 `\\*\CompanyShare` 则本机作客户端，否则认领为文件服务器并建共享；
  - 映射共享盘：把 `\\<文件服务器>\CompanyShare` 映射为 `S:`（可在配置改盘符）；
  - TrustedHosts：设为发现到的对端 IP（仅内网可信环境）；
  - 上报：本机信息写入 `Mgmt$\hosts\<PC>.json`，对端清单写入 `Mgmt$\inventory\<PC>_peers.json`，供 manager 使用；
  - 启用 WinRM，供后续远程管控。

## 7. 安全与运维注意事项

1. **强密码**：管理账号 `itadmin` 运行时现场输入，不落盘。
2. **共享权限**：`CompanyShare` 默认 `Everyone` 完全控制，便于协作但有风险；生产环境建议改为指定安全组（直接改 `New-SmbShare -FullAccess` 参数）。
3. **TrustedHosts**：设为 `discovered` 仅适合封闭内网；边界不清时改为具体管理机地址（改 `company-config.json` 的 `Security.TrustedHosts`）。
4. **RDP 暴露**：RDP 仅内网开放；**切勿**把 3389 映射到公网。
5. **防火墙**：基线开启的是"专用网络"下的规则；离开办公网络时把网络类别切回"公用"。
6. **清单即资产**：`Mgmt$\hosts` 与 `Mgmt$\inventory` 是实时资产与拓扑，可定期归档审计。

---

## 8. 升级路径与替代

- **家庭版 Windows**：不支持 RDP 主机与部分组策略。可升级到专业版，或远程控制改用 RustDesk 等跨平台方案（脚本可加一步静默安装）。
- **远程/多分支办公**：单局域网无法覆盖外网。在边界部署 VPN（WireGuard/OpenVPN）或零信任组网（Tailscale/ZeroTier），让外网设备逻辑进入同一虚拟网，再复用本文全部脚本。
- **规模化集中管控（GPO）**：电脑增多后建议引入 Windows Server + Active Directory 域，用组策略集中下发防火墙/共享/RDP/软件，超越脚本能力。

## 9. 故障排查

| 现象 | 排查 |
|------|------|
| `discover.ps1` 看不到某些电脑 | 目标防火墙拦了 ICMP/445/3389；改用 `Discover` 指定范围或临时放行；确认同网段 |
| 网络邻居看不到其他电脑 | 双方网络类别为"专用"；`Get-NetConnectionProfile`；防火墙"网络发现"已启用 |
| `\\<FS>\CompanyShare` 无法访问 | Server 服务运行；`Get-SmbShare`；用 `itadmin` 凭据 |
| RDP 连不上 | `fDenyTSConnections=0`；Remote Desktop 防火墙规则启用；目标为专业版以上 |
| `manager.ps1` 连不上某机 | 该机已跑 `deploy.ps1`（WinRM 已开）；TrustedHosts 含其 IP；用 `itadmin` 凭据 |
| 心跳/清单未上报 | `\\<FS>\Mgmt$` 可达且 `itadmin` 有写权限 |

## 10. 统一审计溯源、上网管控与网络体检

为满足"安全溯源追责 + 内部网络管控"的要求，在互联基线之上叠加三层能力，且**保持轻量、不引入额外服务器**。

### 10.1 统一审计溯源（追责依据）
所有部署与管控动作经 `lib-audit.ps1` 写入 JSONL 日志，本地与中心双写：
- 本地：`C:\ProgramData\CompanyMgmt\audit\operations.log`
- 中心：`\\<文件服务器>\Mgmt$\audit\operations.log`

每条日志含：时间、操作员、目标、动作、结果。改名动作额外写 `Mgmt$\audit\rename-<新名>.json` 溯源对照表（旧名 / 新名 / IP / MAC / 操作员 / 时间），便于事后核对"谁把哪台机器改成了什么"。

### 10.2 上网管控（可上网 / 可禁用）
`netpolicy.ps1` 以**客户端防火墙**方式实现按主机的"允许 / 禁止上网"：
- **禁止(deny)**：创建/启用出站阻断规则 `Company-NoInternet`，阻塞 TCP 80/443；加 `-HardCut` 可一并阻塞 DNS 53，实现硬断网。
- **允许(allow)**：禁用该规则恢复上网。
- 策略状态写 `Mgmt$\netpolicy\<主机>.json`（含 IP / MAC / 操作员 / 时间），可供溯源；
- `netpolicy.ps1 -ExportGatewayBlacklist <csv>` 导出被禁主机的 **IP/MAC 黑名单**，供路由器/网关侧二次封禁。

> 说明：客户端侧管控在离网或系统重装后可能失效，因此关键终端建议配合网关黑名单形成"客户端 + 网关"双层管控。

管理机批量：
```powershell
.\manager.ps1 -FileServerHost <文件服务器> -NetPolicy deny          # 全网禁止上网
.\manager.ps1 -FileServerHost <文件服务器> -NetPolicy allow         # 全网恢复上网
.\manager.ps1 -FileServerHost <文件服务器> -NetPolicy deny -HardCut  # 硬断网
.\netpolicy.ps1 -Report -FileServerHost <文件服务器>                 # 查看上网策略报表
.\netpolicy.ps1 -ExportGatewayBlacklist .\blacklist.csv -FileServerHost <文件服务器>  # 导出网关黑名单
```

### 10.3 网络体检（只读检查）
`netcheck.ps1` **只读体检，不改任何配置**，检查项：
网络类别、IP 冲突(启发式)、网关可达、DNS 解析、SMB 监听、RDP 监听、监听端口、多网卡(双网卡)、WinRM。
结果写 `Mgmt$\netcheck\<主机>.json`，综合结论分 `健康 / 注意 / 异常`。

管理机批量：
```powershell
.\manager.ps1 -FileServerHost <文件服务器> -NetCheck    # 全网体检并集中上报
.\manager.ps1 -FileServerHost <文件服务器> -NetReport   # 汇总"上网策略"+"网络体检"两张报表
.\netcheck.ps1 -FileServerHost <文件服务器>             # 本机体检并上报
```

### 10.4 额外建议增加的管控能力
结合内部网络管理诉求，建议后续可按需扩展（均复用现有中心清单与 WinRM 通道，不增服务器）：
1. **软件/进程清单与合规**：定期收集已装软件与自启动项，发现违规软件。
2. **USB / 外设管控**：批量禁用可移动存储，防数据外泄。
3. **屏幕/操作审计**：配合合规要求记录操作（需注意隐私与劳动法边界）。
4. **定时策略**：用 Windows 计划任务或本工具的自动化能力，定时跑 `-NetCheck` 与资产收集，形成趋势。
5. **告警**：体检 FAIL（如网关不可达、IP 冲突）时，将结论写入审计并可由管理机集中标红。

## 11. 声明

本方案为工作组模式下的轻量级、自感知互通工具，重在低成本、立即执行、零人工参数搬运。它不替代企业级网络规划（VLAN、AD 域、EDR、备份）。正式规模部署前，建议由网络/安全负责人评审第 7、8、10 节。

## 12. 版本与质量说明（v6：编码与权限强化）

- **消除中文乱码**：所有 `.ps1` / `.json` 已统一为 **UTF-8 带 BOM**；运行时日志/清单/JSON/CSV 输出强制 UTF-8（PowerShell 5.1 与 7 一致）。已用 **PowerShell 5.1 实测**确认中文读取与写出均正常，从根上消除中文在默认 ANSI 下乱码的问题。
- **权限最大化 + 用户决策**：新增 `lib-init.ps1`，入口脚本统一 `dot-source` 后在**非管理员时自动以 RunAs 提权**（仅直接运行触发，被引用时不触发）；提权弹出的系统 UAC 由用户确认，符合"重点是可供用户决策"。
- **全用户覆盖**：部署写入机器级路径 `$env:ProgramData\CompanyMgmt` 与中心隐藏共享 `Mgmt$`，不绑定任何单用户；普通员工也能运行 `collect-pcinfo.ps1` 自动采集本机配置发回。
- **配套教程**：见 `金网通-开发实战教程.html`（可视化演示 + 自研扩展教程，含厦门金奕鸣科技有限公司出品信息与许可声明）。

## 13. 商业授权与上线（闭环）

本方案已从"内部工具"升级为可商用闭环产品，完整说明见 `商业授权与上线说明.md`（同目录）。核心：

- 厂商用 `scripts/gen-license.ps1` 以 RSA-2048 私钥签发 `company.lic`，客户放入 `scripts/` 即激活。
- 版本分层 Free / Trial / Pro / Enterprise，按设备数与功能权益区分；`lib-license.ps1` 在向导/部署/管控启动时校验签名、有效期、设备数上限与功能权益。
- 无授权按 Free 版运行（3 台、仅基础互联）；超额或过期即阻断高级功能并提示升级，形成续费驱动的营收闭环。
- 私钥 `vendor.key.json` 仅厂商持有、已加入 `.gitignore`，绝不入库或随产品分发。

> **发布前必做**：厂商首次发布前运行一次 `gen-license.ps1`，以把真实公钥写入 `lib-license.ps1`（替换占位符），否则客户端验签会失败。
