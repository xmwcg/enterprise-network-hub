二、系统架构（一张图看懂）

中心是一台"文件服务器"（任意一台电脑自动认领），所有电脑通过 WinRM 与管理共享互联；日志与体检结果集中回传中心。

每台电脑只需以管理员运行一次 deploy.ps1；管理机用 manager.ps1 做批量管控。所有数据走既有 Windows 能力，不新增服务器。

三、面向"所有用户"与"每台电脑自动检查"

① 全用户覆盖（机器级，非单用户）

部署写入 $env:ProgramData\CompanyMgmt（机器级，所有登录用户共享）与中心隐藏共享 Mgmt$；管理账号为统一本地管理员，不绑定某个具体人。配置与日志对全网可见、可追溯。

② 每台电脑自动采集配置

collect-pcinfo.ps1 自动读出本机：主机名 / 系统版本 / 是否加域 / IP / 网关 / DNS / 网卡 / 是否支持 RDP 主机。普通员工也能一键运行并把结果发回，无需懂技术。

③ 全网自动发现与清点

discover.ps1 只读扫描子网，列出脚本"看得到"的所有电脑；manager.ps1 -CollectInventory 批量拉取在线状态与开机时间，自动生成资产清单。

④ 每台电脑自动体检

netcheck.ps1 在每台电脑本地执行 9 项只读检查（网络类别/网关/DNS/SMB/RDP/端口/双网卡/WinRM/IP 冲突），结果自动回传中心 Mgmt$\netcheck。

设计原则：采集与检查都是"本地执行、自动回传"，公司只需在每台电脑跑一次脚本，无需人工逐台登记参数。

四、权限最大化，但决策权在你手里

自动提权（权限最大化）

入口脚本统一 dot-source lib-init.ps1，非管理员时自动以 RunAs 重启为管理员（仅限直接运行，被引用时不触发）。因此部署/管控/上网/体检类脚本永远以最高权限执行，避免"权限不足"导致半途失败。

UAC = 用户决策

提权会弹出系统 UAC 让你确认——"是否给此脚本最高权限"由你点"是/否"决定。脚本本身不偷偷提权，符合"重点是可供用户决策"。

编码无乱码（已修复）

所有脚本/配置统一为 UTF-8 带 BOM，运行期输出也强制 UTF-8，从根上消除中文在 PowerShell 5.1 默认 ANSI 下乱码的问题（已用 PS5.1 实测验证）。

全程审计溯源

lib-audit.ps1 把部署、改名、上网管控、批量命令等动作写成 JSONL，含时间/操作员/目标/动作/结果，本地与中心双写，方便公司追责。

# lib-init.ps1（节选）：自动提权 + 强制 UTF-8 输出
if ($wp.IsInRole([Security.Principal.WindowsBuiltinRole]::Administrator)) { return }
Start-Process -FilePath 'powershell.exe' -ArgumentList $sb -Verb RunAs `
             -WorkingDirectory (Split-Path $ScriptPath)
# 文件写入一律 UTF-8（带 BOM，PS5.1/7 一致）
$PSDefaultParameterValues['Out-File:Encoding'] = $enc   # utf8 / utf8BOM

五、五分钟快速开始

  1. 准备：把整套 scripts/ 目录复制到一台电脑，以管理员打开 PowerShell，cd 到该目录。
  2. 配置（用户决策）：运行 .\wizard-gui.ps1（图形）或 .\wizard.ps1（命令行），按中文菜单选配工作组名、管理账号、文件服务器策略、是否自动改名/装 RustDesk 等，可"上一步/下一步"，最后生成 company-config.json。
  3. 预览（可选，只读）：.\discover.ps1 先看脚本能发现哪些电脑，不改任何配置。
  4. 每台电脑部署：在 每一台 电脑上以管理员运行 .\deploy.ps1（会自动提权）。它自动检测本机、采集对端、建立共享、改名、启用 WinRM，并把清单与心跳上报中心。
  5. 集中管控：在管理机运行 .\manager.ps1 -FileServerHost <文件服务器> -CollectInventory 盘点；-NetPolicy deny 全网禁网；-NetCheck 全网体检；-NetReport 汇总。
文件服务器无需单独准备：deploy 会以 AUTO 策略自动认领/复用已有的 CompanyShare，或在子网内首台运行时认领。

六、开发教程（2/3）：核心模块怎么读

模块职责被谁调用
lib-init.ps1设置 UTF-8 输出默认值；定义 Request-AdminOrElevate 自动提权所有入口脚本
lib-discovery.ps1Find-Peers：按子网枚举存活主机与开放端口deploy / discover / manager
lib-audit.ps1Write-Audit：写 JSONL 到本地与中心 Mgmt$\auditdeploy / manager / netpolicy / netcheck
deploy.ps1自动检测→采集→改本机配置→上报；可 dot-source 的库函数 + 主体分离员工/IT 每台运行
manager.ps1读中心清单，批量 WinRM 执行；汇总上网策略与体检报表管理机运行
netpolicy.ps1Set-InternetPolicy：防火墙 allow/deny + 状态入库 + 网关黑名单导出manager / 直接运行 / 引用
netcheck.ps1Invoke-NetCheck：9 项只读体检 + 结果回传manager / 直接运行 / 引用
关键约定：可复用模块（netpolicy / netcheck）用 if ($MyInvocation.InvocationName -ne '.') 区分"被引用（只暴露函数）"与"直接运行（执行主体）"，避免被 manager 引用时误跑主体。

六、开发教程（3/3）：自己加一个功能

示例：新增"USB 存储管控"模块，沿用既有的 编码 / 提权 / 审计 / 中心回传 四件套。复制下面的骨架即可。

# usb-policy.ps1 —— 新功能骨架（沿用项目约定）
. .\lib-init.ps1
. .\lib-audit.ps1
if ($MyInvocation.InvocationName -ne '.') {
    Request-AdminOrElevate -ScriptPath $PSCommandPath -Bound $PSBoundParameters -Unbound $args
}

function Set-UsbPolicy {
    param([ValidateSet('allow','deny')][string]$Action, [string]$FileServerHost)
    # 1) 真正干活（示例：用组策略首选项或 devnode 禁用 USB 存储）
    # 2) 写审计（自带时间/操作员/结果）
    Write-Audit -FileServerHost $FileServerHost -Entry ([ordered]@{
        Target = $env:COMPUTERNAME; Action = "UsbPolicy"; Value = $Action; Result = "OK" })
}

# 主体（仅直接运行时执行）
if ($MyInvocation.InvocationName -ne '.') {
    Set-UsbPolicy -Action $Action -FileServerHost $FileServerHost
}

接入 manager.ps1 做批量

在 manager.ps1 顶部 . .\usb-policy.ps1，新增开关 -UsbPolicy <allow|deny>，在分支里对每台主机调用 Set-UsbPolicy 即可。无需改架构，纯"加文件 + 加一个分支"。

你完全可以加入任何想要的能力：软件合规清单、定时体检计划任务、异常外联告警、打印机/蓝牙管控……模式都是"函数 + 审计 + 可选中心回传"。
